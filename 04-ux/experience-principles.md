# Experience Principles

## Clarity, Confidence, Calm

Every Business X-Ray interaction should reduce confusion rather than add another thing for the owner to manage.

## The Owner's Emotional Journey

The journey should move the user from:

1. vague frustration
2. recognition
3. understanding
4. prioritisation
5. confidence
6. action

## Writing Rules

- Use British English.
- Avoid jargon where plain language works.
- Do not exaggerate certainty.
- Explain assumptions.
- Use active, practical language.
- Prioritise outcomes over features.

## Interface Rules

- Show only what matters next.
- Avoid dashboard overload.
- Make scores explainable.
- Every button should have a clear purpose.
- Every report section should lead to a next step.
- Empty states should teach, not simply say "no data".

## Public Site Experience

The website should feel premium, calm and intelligent.

It should not look like:

- a generic marketing agency
- an SEO landing page farm
- a finance dashboard
- a cheap AI tool

It should feel like:

- a trusted diagnostic experience
- a premium advisory product
- a business intelligence platform
